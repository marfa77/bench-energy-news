# LLM Search Optimization (GEO/AEO) - Implementation Guide

## Overview

This project is optimized for **Large Language Model (LLM) Search**—also known as **Generative Engine Optimisation (GEO)** or **Answer Engine Optimisation (AEO)**. The content is structured to be **retrieved, extracted, and cited** by AI systems like ChatGPT, Perplexity, and Google AI Overviews.

#### AI-Specific Robots.txt
- **Status:** ✅ Implemented

#### Server-Side Rendering (SSR)
- **Status:** ✅ Implemented
- **Technology:** Next.js with Static Site Generation (SSG)
- **Output:** Static HTML files in `out/` directory
- **Benefit:** AI bots can access content without JavaScript execution

#### Schema.org Structured Data
- **Status:** ✅ Implemented
- **Types used:**
  - `NewsArticle` for news posts
  - `BlogPosting` for blog articles
  - `Organization` for Bench Energy
  - `FAQPage` for question-answer pairs
  - `BreadcrumbList` for navigation
- **Format:** JSON-LD (machine-readable)
- **Location:** In `<head>` of every page

### ✅ Phase 2: Content Engineering

#### Answer Capsules
- **Status:** ✅ Implemented
- **Format:** 40-80 word standalone answers
- **Location:** Immediately after `<h1>` title
- **Implementation:** Generated by Claude in `post_versions_generator.py`
- **Example:**
  ```html
  <div class="answer-capsule">
    <p>Coal prices in Australia surged to $245 per tonne in January 2026, 
    driven by supply disruptions from heavy rainfall in Queensland mines. 
    This represents a 12% increase from December 2025 levels.</p>
  </div>
  ```

#### Goldilocks Zone Sections
- **Status:** ✅ Implemented
- **Target:** 120-180 words per section
- **Implementation:** Claude generates content in optimal chunk sizes
- **Benefit:** Improves AI extraction accuracy by up to 35%

#### Explicit Nouns and Numbers
- **Status:** ✅ Implemented
- **Guidelines:**
  - Use specific numbers: "$245 per tonne" not "high prices"
  - Use concrete nouns: "Queensland mines" not "some locations"
  - Avoid vague adjectives: "contains 3g" not "super powerful"
- **Implementation:** Enforced in Claude prompts

## Schema.org Implementation

### NewsArticle Schema

```json
{
  "@context": "https://schema.org",
  "@type": "NewsArticle",
  "headline": "...",
  "abstract": "[Answer Capsule - 40-80 words]",
  "mainEntity": {
    "@type": "FAQPage",
    "mainEntity": [
      {
        "@type": "Question",
        "name": "What is this news about?",
        "acceptedAnswer": {
          "@type": "Answer",
          "text": "..."
        }
      }
    ]
  }
}
```

### FAQPage Schema

- **Purpose:** Makes question-answer pairs explicitly extractable
- **Questions covered:**
  - What is this news about?
  - What is the market impact?
  - What are the price implications?
  - Which regions are affected?
